# Thanks for contributing to this project!

## Ways to contribute
1. Raising issues to report bugs or request features
2. Fixing existing issues (look for the `good first issue` label for easy tasks)
3. Adding new features
4. Improving documentation

## How to contribute
1. Fork the project
2. Create a new branch for your changes (e.g. `fix/bug-description`, `docs/update-readme`, `feat/new-feature`)
3. Write your changes following the code style guidelines
4. Test your changes to ensure they work correctly
5. Create a pull request with: `Closes #<issue-number>` (replace `<issue-number>` with the ID of the issue you're fixing)

## Commit message format
Please use clear and descriptive commit messages following this format:
- `fix: <description>` for bug fixes
- `feat: <description>` for new features
- `docs: <description>` for documentation updates
- `chore: <description>` for maintenance tasks

## Code style
- Follow existing code conventions in the project
- Use 4 spaces for indentation
- Keep lines under 120 characters
- Add comments for complex logic

All contributors welcome :)
